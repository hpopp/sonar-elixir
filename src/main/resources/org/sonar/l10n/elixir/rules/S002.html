Modules that grow too large become difficult to understand, test, and maintain. Consider splitting large modules into smaller, focused modules that each represent a single concept.

Noncompliant Code Example

defmodule MyApp.KitchenSink do
  # 300+ lines of mixed concerns
end

Compliant Solution

defmodule MyApp.Accounts do
  # User account logic
end

defmodule MyApp.Notifications do
  # Notification delivery logic
end

Configuration

The default threshold is 500 lines per module, matching the Erlang community convention (Elvis linter).
